GCP · MLOps · 2026

End-to-End ML Pipeline mit Vertex AI

Vollständige MLOps-Pipeline auf Vertex AI: Random-Forest-Klassifikation zur Notenvorhersage (A–F), orchestriert mit Kubeflow Pipelines (KFP v2) – von der Datenbeschaffung bis zum bedingten Modell-Deployment mit FastAPI-Frontend.

  • Python
  • Vertex AI
  • Kubeflow Pipelines (KFP)
  • FastAPI
  • GCP

Projektzusammenfassung

Im Rahmen des GCP Professional Machine Learning Engineer-Kurses wurde eine vollständige MLOps-Infrastruktur auf Google Cloud Platform aufgebaut. Das Herzstück ist eine automatisierte Vertex AI Pipeline auf Basis von Kubeflow Pipelines (KFP v2), die den gesamten ML-Lebenszyklus reproduzierbar und skalierbar orchestriert. Der Use Case: Klassifikation von Schülerleistungen in Notenstufen A bis F mittels Random Forest.

Die Pipeline besteht aus sechs eigenständigen KFP-Komponenten, die als containerisierte Python-Funktionen definiert sind. Rohdaten werden als CSV aus Google Cloud Storage geladen, bereinigt, gesplittet und als Parquet gespeichert. Das Modelltraining wird direkt als KFP-Komponente ausgeführt und in Vertex AI Experiments geloggt. Nach der Evaluation – Accuracy, F1 und Confusion Matrix – wird das Modell per KFP-Condition nur dann im Vertex AI Model Registry registriert und auf einem Endpoint deployt, wenn definierte Qualitätsschwellen erreicht werden.

Vorhersagen sind über ein FastAPI-Backend mit HTML-Frontend abrufbar. Die gesamte Orchestrierung läuft serverlos auf Vertex AI Pipelines; Artefakte, Metriken und Modell-Lineage werden automatisch in Vertex AI Experiments und ML Metadata gespeichert.

GCP · Cloud Run · 2026

OpenAI Chatbot – Containerisiertes Microservice-Deployment auf GCP

KI-Chatbot mit FastAPI-Backend (LangChain + GPT-3.5-turbo) und Chainlit-Frontend als zwei unabhängige Docker-Container auf GCP Cloud Run – Infrastruktur vollständig als Terraform-IaC mit Secret Manager und Cloud Build CI/CD.

  • Python
  • FastAPI
  • Chainlit
  • LangChain
  • OpenAI API
  • GCP Cloud Run
  • Terraform

Projektzusammenfassung

Ein containerisierter KI-Chatbot mit klarer Trennung von Frontend und Backend als zwei eigenständige Microservices auf Google Cloud Platform. Das FastAPI-Backend übernimmt die gesamte KI-Logik: Eingehende Nutzerprompts werden per LangChain in eine strukturierte Message-Chain überführt – bestehend aus System-Message und Human-Message – und an das GPT-3.5-turbo-Modell von OpenAI übermittelt. Der OpenAI-API-Key wird sicher über den GCP Secret Manager verwaltet und zur Laufzeit als Environment-Variable in den Container injiziert.

Das Chainlit-Frontend stellt eine moderne Chat-Oberfläche bereit, die asynchron per HTTP mit dem Backend kommuniziert. Beide Services laufen als unabhängige Cloud-Run-Instanzen – auto-skalierend und serverlos. Docker-Images werden mit Google Cloud Build gebaut und in der Artifact Registry gespeichert, bevor sie über Cloud Run deployed werden.

Die gesamte Infrastruktur – Cloud-Run-Services, IAM-Bindings, Secret-Referenzen und Artifact-Registry-Repositories – ist vollständig als Terraform-Infrastructure-as-Code definiert. Umgebungsspezifische Konfigurationen für INT, ACC und PROD werden über separate tfvars-Dateien verwaltet.
